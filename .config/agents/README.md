# Agent Layout

Tracked files define global agent behavior and setup entry points:

- `~/.claude/CLAUDE.md` is the Claude Code global instruction entry point.
- `~/.codex/AGENTS.md` is the Codex global instruction entry point.
- `~/.config/agents/common/global.md` contains shared persona and workflow rules.
- `~/.config/agents/skills/README.md` documents the curated global skill setup.
- `~/.config/zsh/agents.zsh` exposes `agentsetup`, `agentskillssetup`, `agentmcpsetup`, and `agentdoctor`.
- `~/scripts/setup-agent-skills.zsh` installs curated skills.
- `~/scripts/setup-agent-mcp.zsh` installs MCP servers.

Local generated state is intentionally untracked:

- `~/.config/agents/mcp.env` is a secret file restored with `dspullall` and updated with `dspush`.
- `~/.codex/auth.json` and `~/.claude.json` are machine-local login/client state.
- `~/.codex/config.toml` may contain generated MCP configuration.
- `~/.agents/skills` and `~/.claude/skills` are generated by the skills setup.

Use `agentsetup` after restoring dotfiles and secrets on a personal machine. Use `agentdoctor` to inspect status without modifying files.
